<task type="auto">
  <name>Task 2: Push docs/ to main and verify live deployment</name>
  <files>docs/index.html</files>
  <action>
1. Ensure all docs/ files from Plan 01 are committed and pushed to main branch.

2. Check GitHub Pages deployment status:
   ```
   gh api repos/{owner}/{repo}/pages --jq '.html_url'
   ```
   This returns the live URL (e.g., https://aarjay.github.io/machine_learning_skill/).

3. Wait up to 60 seconds for deployment. Check status:
   ```
   gh api repos/{owner}/{repo}/pages/builds --jq '.[0].status'
   ```
   Should show "built".

4. Verify the live URL responds:
   ```
   curl -s -o /dev/null -w "%{http_code}" [LIVE_URL]
   ```
   Expect 200.

5. Verify CSS loads (not a 404):
   ```
   curl -s -o /dev/null -w "%{http_code}" [LIVE_URL]/assets/css/style.css
   ```
   Expect 200.

6. Verify at least one image loads:
   ```
   curl -s -o /dev/null -w "%{http_code}" [LIVE_URL]/assets/images/compare-tech-stocks.png
   ```
   Expect 200.

If any asset returns 404, check for root-absolute paths in the HTML. The most common cause is a leading / in href or src attributes.
  </action>
  <verify>
    <automated>gh api repos/{owner}/{repo}/pages --jq '.html_url' 2>/dev/null && echo "Pages configured"</automated>
  </verify>
  <done>Live GitHub Pages URL returns 200 for index.html, style.css, and at least one image. Zero 404s on deployed assets.</done>
</task>
